SEAMCARVE




SeamCarve with Web Workers

I made this to showcase the benefits of using webworkers. When you resize the image without using the worker the animations on the page will freeze because while the image is processing everything else occuring on the main thread, such as animations will be put on hold. Using a worker allows you to offload a task into a separate thread that will return its results on completion while the main thread continues to animate as normal.

Uses features of ECMAScript 6 and works best in Google Chrome
